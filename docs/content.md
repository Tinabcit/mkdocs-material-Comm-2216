The Overview of Project introduces the Toronto Cupcake website and explains why it was chosen for redesign. The current website has a lot of usability issues, so the goal of this project is to improve navigation, make information easier to find, and create a cleaner and more modern design. This section gives a general idea of what the project is about before going into more detail.

The Norman Heuristic – Proposal & UX/UI section looks at the problems in the original website using Norman’s design principles. For example, the navigation is confusing, there is not enough feedback when users take actions, and the layout feels cluttered. This section explains these issues and suggests how they can be improved to make the website easier to use.

The Website User Stories and User Personas section focuses on the people who will use the website. It includes user stories that describe what users want to do, like ordering cupcakes or checking allergen information. It also includes personas such as a wedding planner or a regular customer, which help show different user needs and expectations.

The UX Problem Statement and Scope clearly explains the main problems with the current website and what parts of the site will be redesigned. It helps define the focus of the project so that the design stays organized and does not try to fix everything at once.

The User Flows and Task Flows section shows how users move through the website step by step. It includes different flows like tracking an order, viewing allergen information, and placing a bulk order. These flows help make sure the process is logical and easy to follow, and that users do not get confused while completing tasks.

The Low-Fidelity Wireframes are simple sketches of the website layout. They focus on structure, content placement, and navigation without worrying about colors or styling. This makes it easier to test ideas quickly and make changes before moving to the final design.

The High-Fidelity Designs show the final version of the website with full styling, including colors, fonts, images, and branding. This is where the design looks more realistic and closer to what users would actually see and interact with.

The Usability Testing and Feedback section explains how the design was tested with users. It includes what feedback was received, what issues were found, and what changes were made to improve the design. This step is important to make sure the website actually works well for real users.

The Final Reflection and Improvements section summarizes what was learned during the project. It talks about what worked well, what challenges were faced, and what could be improved in the future. This shows an understanding of the design process and how it can be improved over time.
