    Usability Testing & Findings 👤

    Overview

    This section summarizes the usability testing conducted on the Toronto Cupcake redesign. The purpose of this testing was to observe how users naturally interact with the interface and to identify any areas where users feel confused, lost, or unsure about what to do next. By analyzing user behavior and feedback, we can improve the overall usability, clarity, and flow of the website.

    User Test #1

    Observations

    During the first test, the user immediately clicked on the cart upon entering the homepage. This indicates that the user’s first instinct was to check their order status rather than begin browsing products. After that, the user followed the “already ordered?” link to view their orders. This suggests that the entry point for ordering or browsing is not clearly communicated.

    The user then skipped the cupcake selection process entirely and navigated directly to allergy information and menu items. This behavior shows that users may prioritize important information such as allergies before committing to browsing products.

    Warning

    The "Cupcakes" menu item is unclear, as the user was not sure what page it would lead to. This creates hesitation and slows down navigation.

    Additionally, the user found the box amount selection confusing. They initially assumed that all cupcakes were already included or pre-selected, which shows that the ordering process is not intuitive.

    Note

    This indicates that users need clearer guidance when starting the ordering process, especially for selecting quantities or box sizes.

    User Test #2

    Observations

    In the second test, the user clicked on the Orders page directly from the navigation bar. Similar to the first test, this again shows that users are more focused on checking order status rather than beginning a new order.

    The user skipped over the cupcake browsing section and went directly to look for allergy-related information. However, they were unable to easily find this information on the cupcake page, which caused confusion and slowed down their experience.

    Warning

    The allergy information is not visible enough and should be more prominent, especially since users actively look for it.

    Another issue observed was that the box size selection step was not obvious. The user was unsure about the correct sequence of actions when placing an order.

    Tip

    A clearer and more structured ordering flow should be implemented:

    1. Select box size
    2. Choose cupcakes
    3. Place order

    Note

    These steps should be clearly displayed on the page, such as through headers or a visual progress indicator, to guide users step-by-step.

    General Feedback

    Positive Feedback

    Overall, users found some parts of the interface easy to use. The prompts provided in the design worked well and did not cause major issues during testing. Once users located the allergy-related features, they found them intuitive and helpful. The general layout of the page was also described as clean and “pretty good,” indicating that the foundation of the design is strong.

    Note

    This shows that the design is on the right track, but still needs improvements in clarity and user guidance.

    Issues Identified

    Despite the positive aspects, several usability issues were identified. Users consistently clicked on the cart first, which suggests that the main call-to-action for starting an order is not clear. The bulk order button was also not noticeable enough, meaning users may miss an important feature.

    Warning

    Key actions such as Bulk Order and starting an order are not visually emphasized enough, which affects usability.

    Users also expressed confusion about when to select the order size. Many users expected to choose the box size before viewing the cupcake options, but this step was not clearly presented. Additionally, the allergy information on the cupcake page was not prominent, even though it is an important concern for users.

    The navigation wording also caused minor confusion. For example, using “Order” instead of “Orders” made it less clear that the page contains past or current orders. Lastly, the feedback section appeared somewhat crowded, which could overwhelm users.

    Recommendations

    Navigation Improvements

    To improve navigation, it is important to use clear and familiar labels that match user expectations. The logo should also link back to the homepage, as this is a standard behavior on most websites. Important actions such as viewing the cart or orders should be easy to find and visually distinct.

    Tip

    Improving navigation clarity will help users move through the site more confidently and efficiently.

    Ordering Flow

    The ordering process should be redesigned to follow a clear and logical sequence. Users should be guided step-by-step so they understand what to do at each stage.

    Tip

    Recommended ordering steps:

    1. Step 1: Choose box size
    2. Step 2: Select cupcakes
    3. Step 3: Place order

    Note

    This process should be visually displayed using a progress bar, numbered steps, or section headings to reduce confusion.

    Content & Visibility

    Improving the visibility of important information is essential. Allergy details should be easier to find and possibly highlighted or enlarged on the cupcake page. In addition, unnecessary clutter should be reduced to help users focus on key tasks.

    Tip

    Important buttons such as Bulk Order should be emphasized using size, color, or placement.

    Conclusion

    Note

    The usability testing shows that while the overall design is functional and visually acceptable, users experience confusion with navigation and the ordering process. By improving clarity, simplifying the flow, and making key information more visible, the overall user experience can be significantly enhanced.
